# Plotty Backend

Серверная часть платформы для публикации интерактивных литературных произведений (фанфиков) с интегрированным набором AI-инструментов поддержки авторов (интерактивный редактор, проверка канона, генерация обложек и проверка логики сюжета).

Проект разработан в рамках образовательной программы учебного центра **Технопарк (VK Education)**.

---

## Архитектурные решения

Система спроектирована с разделением на два ключевых сервиса для изоляции веб-трафика от тяжелых фоновых вычислений.

- **Core API (Go):** Обслуживает клиентские HTTP-запросы, отвечает за авторизацию пользователей, управление правами доступа, хранение метаданных произведений, транзакции баланса и обработку платежей.
- **ML Worker (Go):** Фоновый обработчик очередей. Изолирует ресурсоемкие операции искусственного интеллекта (генерация изображений, анализ текста, разбор лора глав) от основного потока запросов API. Общение между сервисами реализовано асинхронно через **RabbitMQ**.
- **Базы данных и хранилища:**
  - **PostgreSQL:** База данных Core API для хранения реляционной структуры.
  - **PostgreSQL + pgvector:** Хранилище векторов для ML-сервиса, использующее расширение `pgvector` для работы с векторными представлениями текстов.
  - **Redis:** Высокопроизводительное in-memory хранилище сессий пользователей.
  - **MinIO (S3):** Объектное хранилище для аватаров пользователей и сгенерированных ИИ обложек.

## Основные возможности платформы

### 1. Жизненный цикл произведений и черновики

- **Многоэтапное редактирование:** Авторы могут сохранять изменения в черновики глав, публиковать их или полностью откатывать изменения до состояния последней опубликованной версии.
- **Аналитика просмотров:** Система фиксирует и отображает детальную статистику уникальных просмотров по главам для каждого произведения автора.

### 2. Асинхронный ИИ-помощник автора

- **Анализ логики сюжета:** Сравнивает черновик новой главы со структурированным лором всех предыдущих глав произведения для выявления сюжетных нестыковок и логических дыр.
- **Проверка на соответствие канону:** Анализирует черновик на соответствие правилам выбранного вымышленного мира. Извлекает упомянутых персонажей, ищет по ним факты в базе знаний фэндома и отправляет контекст в LLM для выявления явных противоречий.
- **Генерация обложек:** На основе текста главы улучшает пользовательский промпт, генерирует иллюстрацию через API Gigachat, скачивает её, сохраняет в S3-хранилище и привязывает к главе произведения.
- **Автогенерация аннотаций:** На основе первой главы произведения автоматически генерирует краткое интригующее описание истории для карточки произведения.

### 3. Интерактивное взаимодействие читателей

- **Полочная система:** Возможность добавлять произведения в персональную библиотеку со статусами («читаю», «в планах», «прочитано», «брошено», «любимое»).
- **Пользовательские коллекции:** Читатели могут объединять опубликованные произведения в кастомные тематические подборки.
- **Обратная связь:** Система лайков произведений и древовидная структура комментариев к главам с возможностью редактирования и удаления.

### 4. Биллинг и монетизация

- **Система AI-кредитов:** Генерация обложек и проверки текста расходуют внутренние кредиты пользователя.
- **Покупка пакетов:** Поддержка платежных пакетов кредитов с интеграцией платежного шлюза YooMoney.

## Стек технологий

- **Backend Core & Worker:** Go 1.26, pgx (v5), go-redis (v9), minio-go (v7), Gorilla Mux, Viper.
- **ML & NLP:** Python 3.11, FastAPI, SentenceTransformers, PyTorch, Uvicorn.
- **Инфраструктура:** PostgreSQL 15 (+ pgvector), Redis 7, RabbitMQ 3, MinIO S3, LanguageTool API, Docker & Docker Compose, Traefik Proxy.
- **Интеграция ИИ:** API GigaChat.

## Локальное окружение и запуск

Для развертывания проекта в локальной среде разработки необходим установленный Docker и Docker Compose.

1. Создайте файл конфигурации `.env` на основе примера:

   ```bash
   cp .env.example .env
   ```

   _Заполните обязательные переменные, включая GIGACHAT_AUTH_KEY для работы функций искусственного интеллекта._

2. Запустите инфраструктурные сервисы и приложения:

   ```bash
   make up
   ```

   _Команда развернет базы данных, очереди сообщений, объектное хранилище, а также запустит Core API и ML Worker в фоновом режиме._

3. Для пересборки контейнеров после изменения исходного кода используйте:

   ```bash
   make build
   ```

4. Остановка системы:

   ```bash
   make down
   ```
